Classroom Management Api (Zarządzanie Salami Api)


Requests Routes:

    Buildings

  • api/buildings
  • api/buildings/{id}

    • Classroom functions

  • api/classroomFunctions
  • apiclassroomFunctions/{id}

    • Campus

  • api/campus
  • api/campus/{id}

    • Classroom structures

  • api/classroomStructures
  • api/classroomStructures/{id}

    • Classrooms

  • api/classrooms (gets all classrooms together with educationalClassrooms parameters (for all classrooms)
  • api/classrooms?Buildings=..&TV=..&Projector=..&AirConditioning=..&OnlyEducationalClassrooms=..&ClassroomFunctions=..&AccessForTheDisabled=.. &SizeMin=..&SizeMax=..&PlacesMin=..&PlacesMax=.. - Parameters: Buildings, ClassroomFunctions can have multiple values (FILTERING)
  • api/classrooms/{id} [PUT] computer id param in body of request (for binding a classroom with a computer)
  • api/classrooms [POST] (creating a classroom, together with virtual machines and software, don't have to specify an id of a computer)
  • api/classrooms [PUT] (updating a classroom, if we want to update an educationalClassroom then we have to add a parameter CzyDydaktyczna=true)

    • Educational Classrooms

  • api/educationalClassroms
  • api/educationalClassroms/{id}

    • Computers

  • api/computers
  • api/computers [POST]
  • api/computers&idSala=... [POST] (this one is for adding a computer + if we want, updating a classroom, so that we have the newly created computer assigned immediately to the classroom)
  • api/computers [PUT]
  • api/computers/{id} [PUT] (for adding a computer to a classroom, we give in the body of request an id of a computer)
  • api/computers/{id}

    • Virtual Machines

  • api/virtualMachines
  • api/virtualMachines/{id}

    • VirtualMachineComputers

  • api/virtualMachineComputers

    • Monitors

  • api/monitors
  • api/monitors/{id}

    • Software

  • api/software
  • api/software/{id}

    • ComputerSoftware

  • api/computerSoftware

    • Floors

  • api/floors (returns a list of all distinct floors)
